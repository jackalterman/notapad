## Performance

### E1 — Debounce highlight scheduling 🔴 ~~🟢~~
**Status:** `[x]` FIXED 2026-04-12 (as part of Bug B3)
**File:** `notapad.py · _schedule_highlight()`
See also: Bug B3. The fix is a one-liner. After fixing the bug, consider raising the
debounce to 150ms for files over 50k characters.

### E2 — Cache search index; invalidate on text change only 🟠
**Status:** `[x]` DONE 2026-04-12
`_text_version` counter added to `__init__`, incremented via `<<Modified>>` event. `_do_search()` stores `_search_cache_key = (term, case, regex, whole, text_version)` and early-exits if the key is unchanged. F3/find_prev on an unchanged buffer costs zero rescans.

### E3 — Large file guard: disable highlight + gutter above threshold 🟠
**Status:** `[x]` DONE 2026-04-12
`_redraw_gutter()` checks char count via `text.count()` and early-exits above 300k chars. `_schedule_highlight()` does the same and updates `status_lang` to show "⚡ Large file" instead of the language name. Both checks reset transparently when content drops back below threshold.

### E4 — Batch gutter redraws with a single position pass 🟡
**File:** `notapad.py · _redraw_gutter()`
**Status:** `[x]` DONE 2026-04-13
`_redraw_gutter()` replaced with a pixel-probe loop: steps `y` from 0 to viewport height by `font.metrics("linespace")` pixels. At each step calls `text.index("@0,{y}")` (single Tcl call) to get the logical line number, then `text.bbox("@0,{y}")` to get the exact pixel row for drawing. A `drawn` set prevents duplicate labels when word-wrap causes multiple visual rows per logical line. Eliminates the old `dlineinfo(f"{ln}.0")` call per logical line and the initial first/last range probe, reducing Tcl round-trips to O(visible visual rows).

### E5 — Debounce settings saves 🟠 ~~🟢~~
**Status:** `[x]` DONE 2026-04-12
`SettingsManager.set()` now schedules a `threading.Timer(2.0)` instead of writing immediately. `save_immediate()` added — called from `on_close()` to guarantee a final flush before the process exits. Timer is daemonised so it can't block exit.

### E6 — Incremental dirty-region highlighting on edit 🟡
**File:** `notapad_app/editor.py · apply_highlight()`

Instead of rescanning a fixed ±40-line window from the viewport top on every highlight
cycle, track which line was last edited (via `<<Modified>>` + `text.index(INSERT)`) and
re-highlight only from the start of that line to the nearest block-comment boundary below.
This matches Sublime Text's incremental lexer approach and eliminates redundant
re-highlighting of lines the user hasn't touched.

## Settings & Session

### E7 — Remember last open file and cursor position 🟠 ~~🟢~~
**Status:** `[x]` DONE 2026-04-12
`on_close()` persists `last_file` and `last_cursor` then calls `save_immediate()`. `main()` restores both on launch when no CLI path is provided; `os.path.isfile()` guards against deleted/moved files. New defaults added to `SettingsManager`.

### E8 — Window geometry persistence ~~🟢~~
**Status:** `[x]` DONE 2026-04-12
`on_close()` saves `wm_geometry()` to settings. `__init__` applies it immediately after `SettingsManager` is constructed, overriding the hardcoded `980x700` default. Wrapped in `try/except` to handle corrupted geometry strings gracefully.

### E9 — Recent files menu (MRU) 🟡
**Status:** `[x]` DONE 2026-04-12
`recent_files: []` added to `SettingsManager` defaults (max 10 paths). `_push_recent()` prepends the path and deduplicates. File menu gains a "Recent Files ▶" submenu via `AntiqueMenu`, rebuilt on every `open_file()` / `save_as_file()`. Non-existent paths are pruned on launch in `main()`.

## Search

### E10 — Inline find bar (replace Find Toplevel) 🟠
**Status:** `[x]` DONE 2026-04-12
Sublime-style inline `tk.Frame` packed above the status bar. Ctrl+F shows it; Escape hides it and returns focus to the editor. Contains: search entry, ◀/▶ nav buttons, match count, Case/Regex/Word checkboxes. Live-searches as you type (200ms debounce). `search_count_lbl` wired to the bar's count label. Old Toplevel `open_find` retained for any code paths that call it directly.

### E11 — Wrap-around indicator ~~🟢~~
**Status:** `[x]` DONE 2026-04-12
`find_next`/`find_prev` now detect when a wrap has occurred and flash "↩ Wrapped" in the match count label for 1.2 s before restoring the normal "Match N of M" text.

### E12 — Replace within selection 🟡
**File:** `notapad_app/dialogs.py · open_replace()`
**Status:** `[x]` DONE 2026-04-13
Added `in_sel_var = tk.BooleanVar()` checkbox ("In selection") as column 3 of the options row in the Replace dialog. Pre-ticked when the dialog opens with an active text selection. `do_replace_all` has a dedicated in-selection branch: captures `SEL_FIRST`/`SEL_LAST` before the operation (warns if no selection), extracts the substring, runs `re.findall`/`re.sub` on it, then deletes and re-inserts the range bracketed by `edit_separator`/`autoseparators=False` for single-step undo. `do_replace` (single replace) guards: skips replacement if the current match falls outside the selection range. Full-document path is unchanged when the checkbox is unchecked. Helper `_build_pattern()` extracted to share pattern-building logic between both functions.

### E13 — Passive word highlight under cursor 🟡
**Status:** `[x]` DONE 2026-04-12
`word_hi` tag defined in `apply_theme()` (subtle warm tint, distinct from `search_hi`). `_schedule_word_highlight()` debounces at 400ms. `_do_word_highlight()` reads the word under the cursor using `word start`/`word end` indices, guards against non-word chars and active selections, then scans the viewport ±20 lines for matches using `text.search()` in a loop. Cleared by `_clear_word_highlight()` which is called from `_on_key()` and `close_find_bar()`.

## Syntax & Language

### E14 — Language override click on status bar 🟠 ~~🟢~~
**Status:** `[x]` DONE 2026-04-12
`status_lang` now has `cursor=hand2` and a `<Button-1>` binding to `_open_language_menu`. That method builds an `AntiqueMenu` from all `LANG_LABEL` entries and wires each item to `set_language()`.

### E15 — TypeScript dedicated syntax definition 🟡 ~~🟢~~
**Status:** `[x]` DONE 2026-04-12
`"typescript"` entry added to `P` in `config.py`: full JS patterns plus TS-only keywords (`interface`, `type`, `enum`, `namespace`, `keyof`, `infer`, `readonly`, `abstract`, `declare`, `implements`, `satisfies`, `override`, `as`). Added to `LANG_LABEL`. `.ts`/`.tsx` remapped in `EXT_LANG`.

### E16 — Bracket and quote matching highlight 🟡
**Status:** `[x]` DONE 2026-04-12
`bracket_match` tag defined in `apply_theme()` (underline + subtle bg tint). `_schedule_bracket_match()` debounces at 80 ms via `after()`. `_do_bracket_match()` probes the character at and around the cursor, finds `([{` / `}])` pairs by walking the text with nesting-aware counting, and tags both positions. Cleared on every cursor move.

## Encoding & Line Endings

### E17 — Detect and display line endings 🟡
**Status:** `[x]` DONE 2026-04-12
`open_file()` now reads raw bytes to detect `\r\n`/`\r`/`\n` before decoding; stores in `self._line_ending`. New `status_eol` label in status bar shows `CRLF`, `CR`, or `LF`. `save_file()` re-encodes with the original line ending. `new_file()` resets to `LF`.

### E18 — Encoding detection 🟡
**Status:** `[x]` DONE 2026-04-12
`open_file()` tries UTF-8 first, then attempts `chardet` if available, then falls back to `latin-1` (which never errors). Detected encoding stored in `self._encoding` and shown in `status_enc`. `save_file()` writes in `self._encoding`. `new_file()` resets to `"utf-8"`.

## Quality of Life

### E21 — Auto-indent on Enter 🟡
**Status:** `[x]` DONE 2026-04-12
`<Return>` bound to `_auto_indent()`: reads leading whitespace of the current line, inserts `\n` + that indent, and returns `"break"` to suppress the default newline. Works for both spaces and tabs.

### E22 — Tab inserts spaces (configurable) 🟡
**Status:** `[x]` DONE 2026-04-12
`tab_size` (default 4) and `use_spaces` (default True) added to `SettingsManager`. `<Tab>` bound to `_handle_tab()`: inserts spaces to the next tab stop when `use_spaces` is True, else literal `\t`. `<Shift-Tab>` bound to `_handle_shift_tab()`: strips up to `tab_size` leading spaces from the current line. Format menu gains a Tab Size submenu (2 / 4 / 8 / Tab character).

### E23 — File change detection on focus 🟡
**Status:** `[x]` DONE 2026-04-12
`_file_mtime` stored at open/save. `<FocusIn>` on root calls `_check_file_changed()`. If mtime has changed a non-blocking `_reload_bar` slides in (styled like the find bar) with Reload and Dismiss buttons. Reload re-reads the file and preserves cursor position. Dismissed bar stays hidden until the next external change.

### E24 — Drag-and-drop file open 🟡 ~~🟢~~
**Status:** `[x]` DONE 2026-04-12
`tkinterdnd2` imported at startup with a silent `ImportError` fallback. When available, root is a `TkinterDnD.Tk` and `<Drop>` is bound to `_on_drop()` which strips Tcl list braces, resolves the path, and calls `open_file()`. When unavailable the app works normally with no drag-and-drop support.

### E25 — Selection-based word highlight 🟡
**File:** `notapad.py · _do_word_highlight()`
**Status:** `[~]` in progress

When the user selects text (mouse drag, double-click, Shift+Arrow), all identical occurrences of the selected string in the viewport should be highlighted — the same `word_hi` tint already used by E13's cursor-idle path.

Behaviour rules:
- Selection must be non-empty, single-line, and ≤ 200 chars to trigger
- If the selected text is a pure identifier (`^\w+$`): use whole-word (`\b...\b`) matching
- Otherwise (selected a snippet with punctuation): use exact literal matching
- The selected text itself shows the built-in `sel` tag highlight; the `word_hi` tag on it is visually covered by `sel` — correct UX
- Cleared on any keypress (existing `_clear_word_highlight()` path) and rescheduled after 400 ms idle
- Implementation: remove the early-exit for active selections in `_do_word_highlight()` and replace it with logic that reads `SEL_FIRST:SEL_LAST` as the highlight word when a valid selection exists, falling through to the cursor-word path (E13) when there is no selection

### E26 — Adjustable double-click word boundaries 🟡
**File:** `notapad.py · _bind_events()`, `_build_native_menu()`, `notapad_app/settings.py`
**Status:** `[~]` in progress

Tkinter's default `<Double-Button-1>` selects using Tcl's `wordchars` definition, which includes alphanumeric + underscore. This is often wrong for CSS (`background-color`), file paths, or any identifier containing hyphens or dots.

Implementation:
- `word_chars_extra` setting (default `""`) — extra characters beyond `[a-zA-Z0-9_]` that count as word chars for double-click
- Override `<Double-Button-1>` with `_handle_double_click()`: clicks on a word character walk left/right using the custom pattern and set the selection, returning `"break"` to suppress Tk's default; non-word-char clicks fall through to Tk default
- `_set_word_chars(extra)` method stores the extra chars in `self._word_chars_extra` and persists to settings
- Format menu gets four presets after the Tab Size items (sep + Standard / +Hyphens / +Dots / +Hyphens & Dots)
