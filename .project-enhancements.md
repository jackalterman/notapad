## Performance

### E1 вЂ” Debounce highlight scheduling рџ”ө ~~рџџұ~~
**Status:** `[x]` FIXED 2026-04-12 (as part of Bug B3)
**File:** `notapad.py В· _schedule_highlight()`
See also: Bug B3. The fix is a one-liner. After fixing the bug, consider raising the
debounce to 150ms for files over 50k characters.

### E2 вЂ” Cache search index; invalidate on text change only рџџ 
**Status:** `[x]` DONE 2026-04-12
`_text_version` counter added to `__init__`, incremented via `<<Modified>>` event. `_do_search()` stores `_search_cache_key = (term, case, regex, whole, text_version)` and early-exits if the key is unchanged. F3/find_prev on an unchanged buffer costs zero rescans.

### E3 вЂ” Large file guard: disable highlight + gutter above threshold рџџ 
**Status:** `[x]` DONE 2026-04-12
`_redraw_gutter()` checks char count via `text.count()` and early-exits above 300k chars. `_schedule_highlight()` does the same and updates `status_lang` to show "вљҰ Large file" instead of the language name. Both checks reset transparently when content drops back below threshold.

### E4 вЂ” Batch gutter redraws with a single position pass рџџҰ
**File:** `notapad.py В· _redraw_gutter()`
**Status:** `[x]` DONE 2026-04-13
`_redraw_gutter()` replaced with a pixel-probe loop: steps `y` from 0 to viewport height by `font.metrics("linespace")` pixels. At each step calls `text.index("@0,{y}")` (single Tcl call) to get the logical line number, then `text.bbox("@0,{y}")` to get the exact pixel row for drawing. A `drawn` set prevents duplicate labels when word-wrap causes multiple visual rows per logical line. Eliminates the old `dlineinfo(f"{ln}.0")` call per logical line and the initial first/last range probe, reducing Tcl round-trips to O(visible visual rows).

### E5 вЂ” Debounce settings saves рџџ  ~~рџџұ~~
**Status:** `[x]` DONE 2026-04-12
`SettingsManager.set()` now schedules a `threading.Timer(2.0)` instead of writing immediately. `save_immediate()` added вЂ” called from `on_close()` to guarantee a final flush before the process exits. Timer is daemonised so it can't block exit.

### E6 вЂ” Incremental dirty-region highlighting on edit рџџҰ
**File:** `notapad_app/editor.py В· apply_highlight()`

Instead of rescanning a fixed В±40-line window from the viewport top on every highlight
cycle, track which line was last edited (via `<<Modified>>` + `text.index(INSERT)`) and
re-highlight only from the start of that line to the nearest block-comment boundary below.
This matches Sublime Text's incremental lexer approach and eliminates redundant
re-highlighting of lines the user hasn't touched.

## Search

### E10 вЂ” Inline find bar (replace Find Toplevel) рџџ 
**Status:** `[x]` DONE 2026-04-12
Sublime-style inline `tk.Frame` packed above the status bar. Ctrl+F shows it; Escape hides it and returns focus to the editor. Contains: search entry, в—Ђ/в–¶ nav buttons, match count, Case/Regex/Word checkboxes. Live-searches as you type (200ms debounce). `search_count_lbl` wired to the bar's count label. ### E11 вЂ” Wrap-around indicator ~~рџџұ~~
**Status:** `[x]` DONE 2026-04-12
`find_next`/`find_prev` now detect when a wrap has occurred and flash "в†© Wrapped" in the match count label for 1.2 s before restoring the normal "Match N of M" text.

### E12 вЂ” Replace within selection рџџҰ
**File:** `notapad_app/dialogs.py В· open_replace()`
**Status:** `[x]` DONE 2026-04-13
Added `in_sel_var = tk.BooleanVar()` checkbox ("In selection") as column 3 of the options row in the Replace dialog. Pre-ticked when the dialog opens with an active text selection. `do_replace_all` has a dedicated in-selection branch: captures `SEL_FIRST`/`SEL_LAST` before the operation (warns if no selection), extracts the substring, runs `re.findall`/`re.sub` on it, then deletes and re-inserts the range bracketed by `edit_separator`/`autoseparators=False` for single-step undo. `do_replace` (single replace) guards: skips replacement if the current match falls outside the selection range. ### E13 вЂ” Passive word highlight under cursor рџџҰ
**Status:** `[x]` DONE 2026-04-12
`word_hi` tag defined in `apply_theme()` (subtle warm tint, distinct from `search_hi`). `_schedule_word_highlight()` debounces at 400ms. `_do_word_highlight()` reads the word under the cursor using `word start`/`word end` indices, guards against non-word chars and active selections, then scans the viewport В±20 lines for matches using `text.search()` in a loop. Cleared by `_clear_word_highlight()` which is called from `_on_key()` and `close_find_bar()`.

## Quality of Life

### E21 вЂ” Auto-indent on Enter рџџҰ
**Status:** `[x]` DONE 2026-04-12
`<Return>` bound to `_auto_indent()`: reads leading whitespace of the current line, inserts `\n` + that indent, and returns `"break"` to suppress the default newline. Works for both spaces and tabs.

### E22 вЂ” Tab inserts spaces (configurable) рџџҰ
**Status:** `[x]` DONE 2026-04-12
`tab_size` (default 4) and `use_spaces` (default True) added to `SettingsManager`. `<Tab>` bound to `_handle_tab()`: inserts spaces to the next tab stop when `use_spaces` is True, else literal `\t`. `<Shift-Tab>` bound to `_handle_shift_tab()`: strips up to `tab_size` leading spaces from the current line. Format menu gains a Tab Size submenu (2 / 4 / 8 / Tab character).

### E23 вЂ” File change detection on focus рџџҰ
**Status:** `[x]` DONE 2026-04-12
`_file_mtime` stored at open/save. `<FocusIn>` on root calls `_check_file_changed()`. If mtime has changed a non-blocking `_reload_bar` slides in (styled like the find bar) with Reload and Dismiss buttons. Reload re-reads the file and preserves cursor position. Dismissed bar stays hidden until the next external change.

### E24 вЂ” Drag-and-drop file open рџџҰ ~~рџџұ~~
**Status:** `[x]` DONE 2026-04-12
`tkinterdnd2` imported at startup with a silent `ImportError` fallback. When available, root is a `TkinterDnD.Tk` and `<Drop>` is bound to `_on_drop()` which strips Tcl list braces, resolves the path, and calls `open_file()`. When unavailable the app works normally with no drag-and-drop support.

### E25 вЂ” Selection-based word highlight рџџҰ
**Status:** `[x]` DONE (merged into `_do_word_highlight()`)
When the user selects text (mouse drag, double-click, Shift+Arrow), all identical occurrences of the selected string in the viewport are highlighted with the same `word_hi` tint as E13's cursor-idle path. Selection must be non-empty, single-line, в‰¤200 chars. Pure identifiers use whole-word matching; other selections use exact literal matching.

### E26 вЂ” Adjustable double-click word boundaries рџџҰ
**Status:** `[x]` DONE
`word_chars_extra` setting (default `""`) вЂ” extra characters beyond `[a-zA-Z0-9_]` for double-click word selection. `_handle_double_click()` overrides `<Double-Button-1>` using the custom pattern. Format menu gains four presets: Standard / +Hyphens / +Dots / +Hyphens & Dots.

`self.search_dir = tk.StringVar(value="down")` is only ever referenced inside the `open_find()` dialog function in `dialogs.py`, which is itself dead code (never called вЂ” superseded by the inline find bar). No live code path reads this variable.
**Fix:** Remove `self.search_dir` from `__init__` and delete the dead `open_find()` function from `dialogs.py`.

### M6 вЂ” font.Font objects created anew on every theme/zoom change рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad.py` `apply_theme()` and `_apply_font()` ~line 618
Both methods create a new `font.Font(font=self._current_font, weight=w)` for every syntax tag (~11 total) on every theme change or zoom step. These accumulate as garbage and add unnecessary overhead during rapid zoom interactions.
**Fix:** Cache two `font.Font` instances (`self._font_normal`, `self._font_bold`) derived from `_current_font`. Reconfigure them with `.config()` instead of creating new instances.

### M7 вЂ” _apply_initial_settings() triggers 3 spurious debounced saves on startup рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad.py` ~line 590
`_apply_initial_settings()` calls `toggle_wrap()`, `_toggle_line_numbers()`, and `toggle_statusbar()`, each of which calls `self.settings.set(...)` вЂ” scheduling three 2-second timer threads for values that haven't changed from what was just loaded.
**Fix:** Add an `_initializing: bool` guard in `__init__` and check it in `settings.set()`, or pass a `persist=False` kwarg to the toggle methods during init.

### M8 вЂ” apply_theme() hardcodes THEMES["light"] as the tag-name source рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad.py` ~line 155
`for tag in THEMES["light"]["syn"].keys():` is used to determine which syntax tags exist. If a future theme adds or renames keys, those tags silently go unprocessed for other themes.
**Fix:** Define `SYN_TAGS = ("comment", "string", "keyword", ...)` as a constant in `config.py` and iterate that instead.

### M9 вЂ” _bm_after and _wh_after not declared in __init__ рџџұ
**Status:** `[ ]` OPEN
**File:** `notapad.py` ~line 891
`_bm_after` and `_wh_after` are first assigned inside `_schedule_bracket_match()` and `_schedule_word_highlight()` respectively, guarded with `hasattr()`. This makes `__init__` an incomplete description of the object's state and misleads static analysis tools.
**Fix:** Add `self._bm_after = None` and `self._wh_after = None` to `__init__`.

### M10 вЂ” SettingsManager config dict is not thread-safe рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad_app/settings.py` ~line 54
`_fire_save()` runs on a `daemon=True` timer thread and reads `self.config` while the main thread may be writing to it. The GIL prevents torn reads on individual dict operations but not on compound sequences.
**Fix:** Add a `threading.Lock` (`self._lock`) around all reads and writes to `self.config`. Alternatively, replace `threading.Timer` with `self.root.after(2000, ...)` to keep all disk I/O on the main thread, eliminating the threading concern entirely.

### M11 вЂ” goto_line() silently ignores non-numeric input рџџұ
**Status:** `[ ]` OPEN
**File:** `notapad_app/dialogs.py` ~line 238
`do_go()` catches `ValueError` with a bare `pass`. If the user types letters and presses Enter, nothing happens вЂ” no validation message, no visual feedback.
**Fix:** Replace `except ValueError: pass` with:
```python
except ValueError:
    messagebox.showwarning(APP_NAME, "Please enter a valid line number.", parent=top)
```

### M12 вЂ” No timeout or ReDoS guard on user-supplied regex рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad.py` `_do_search()` ~line 640; `dialogs.py` `do_replace_all()`
A pathological regex such as `(a+)+$` will hang the UI indefinitely when `search_regex=True`. The Tcl-level `text.search()` call will block the main thread until it completes (or the OS kills the process). The Python-side `re` operations in `do_replace_all()` have the same exposure.
**Fix:** Run regex validation with `re.compile()` in a try/except first to catch `re.error`. For the hangs, consider using `re` with a `threading.Timer` cancellation pattern, or at minimum document the known risk. As a practical guard, warn users when the regex contains repetition constructs likely to cause catastrophic backtracking (`(?:...)+`, `(.+)+`, etc.).

### M13 вЂ” save_as_file() offers only .txt in the type filter рџџұ
**Status:** `[ ]` OPEN
**File:** `notapad.py` ~line 248
`save_as_file()` shows only `("Text Files", "*.txt")` and `("All Files", "*.*")` in the save dialog, even though the app opens and highlights 18 file types. Users saving a `.py` file must manually type the extension or use "All Files".
**Fix:** Expand the filter list to include the most common types (`.py`, `.js`, `.ts`, `.json`, `.html`, `.css`, `.md`, etc.) or simply pre-populate `defaultextension` from the current file's detected language.

### M14 вЂ” select_all() moves cursor to beginning instead of end рџџұ
**Status:** `[ ]` OPEN
**File:** `notapad.py` ~line 766
`select_all()` sets `INSERT` to `"1.0"` (beginning of document). Every other major editor (VS Code, Notepad++, Sublime) places the cursor at `END` after Select All so the next keystroke naturally replaces the entire selection.
**Fix:** Change `self.text.mark_set(tk.INSERT, "1.0")` to `self.text.mark_set(tk.INSERT, tk.END)`.

### M15 вЂ” Bare except: in apply_windows_title_bar() рџџұ
**Status:** `[ ]` OPEN
**File:** `notapad_app/ui_engine.py` ~line 17
```python
except:
    pass
```
This catches `BaseException` including `KeyboardInterrupt` and `SystemExit`.
**Fix:** Change to `except Exception: pass`.

### L5 вЂ” show_appearance_submenu() hardcodes menu_buttons[3] рџџұ
**Status:** `[ ]` OPEN

```python
btn = app.menu_buttons[3] if len(app.menu_buttons) > 3 else app.menu_buttons[0]
```
This hardcodes the View button index. If the menu order changes, the appearance submenu anchors to the wrong button.
**Fix:** Store button references in a `dict` keyed by label in `notapad.py` (e.g. `self.menu_button_map = {"File": btn, ...}`), and look up by name in `dialogs.py`.

### L6 вЂ” pyinstaller is in requirements.txt alongside runtime deps рџџұ
**Status:** `[ ]` OPEN
**File:** `requirements.txt`
`pyinstaller` is a build/packaging tool. New contributors installing `pip install -r requirements.txt` for runtime use will get it unnecessarily.
**Fix:** Move to a `requirements-build.txt` or `requirements-dev.txt` file.

### L7 вЂ” No version pins in requirements.txt рџџҰ
**Status:** `[ ]` OPEN
**File:** `requirements.txt`
All three dependencies (`chardet`, `tkinterdnd2`, `pyinstaller`) are unpinned. A future breaking release of any of them will silently break the app.
**Fix:** Pin to known-good version ranges, e.g. `chardet>=5.0,<6`, `tkinterdnd2>=0.3,<1`.

### L8 вЂ” AntiqueMenu has no keyboard navigation рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad_app/ui_engine.py`
The custom menu is purely mouse-driven. Arrow keys, Tab, and Enter do not work, failing basic accessibility requirements and diverging from expected menu UX.
**Fix:** Implement `<Up>`/`<Down>` traversal between menu items and `<Return>`/`<space>` to activate the focused item. Requires tracking a focused-item index in `AntiqueMenu`.

### L9 вЂ” _on_app_click widget traversal can fail on destroyed widgets рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad.py` ~line 83
The parent-traversal loop calls `self.root.nametowidget(parent.winfo_parent())` without first checking whether `parent.winfo_exists()`. If a widget is destroyed mid-traversal (race with animation or dialog close), this raises `KeyError` вЂ” silently swallowed, but the menu is left open.
**Fix:** Add `if not parent.winfo_exists(): break` at the top of the loop body.

### L10 вЂ” Ctrl+Shift+F binding may be inconsistent across keyboard layouts рџџҰ
**Status:** `[ ]` OPEN
**File:** `notapad.py` ~line 330
`<Control-F>` in Tk means Ctrl+Shift+F on most keyboards (capital F requires Shift). On some international layouts this does not hold. The menu shows "Ctrl+Shift+F" as the accelerator.
**Fix:** Test on at least one non-US keyboard layout. Consider binding both `<Control-Shift-F>` (explicit) and `<Control-F>` to guarantee the shortcut works everywhere.

## Architectural Recommendations (A-series) вЂ” identified 2026-04-15

### A1 вЂ” Break up the 1,600-line monolith
**File:** `notapad.py`
The `Notapad` class owns file I/O, search, bracket matching, word highlighting, syntax scheduling, zoom, session restore, drag-and-drop, and all event bindings. Suggested extraction:
- `notapad_app/file_io.py` вЂ” `open_file`, `save_file`, `_load_large_file_chunked`, encoding/EOL logic
- `notapad_app/search.py` вЂ” `_do_search`, `find_next`, `find_prev`, `find_all`, cache management
- Bracket matching and word highlight already partially live in `editor.py` вЂ” extend it

### A2 вЂ” Introduce a Controller / mediator object between dialogs and app internals
**File:** `notapad_app/dialogs.py`
Dialogs currently receive the entire `app` object and reach deep into its internals
(`app._match_positions`, `app._match_current`, `app.text`, `app._do_search`, etc.).
This creates tight coupling that makes refactoring any single part break dialogs unexpectedly.
A minimal `SearchController` interface (or even just a `SimpleNamespace` of the methods
dialogs need) would decouple them from the implementation.

### A3 вЂ” Replace AntiqueMenu with native Tk Menu widgets
**File:** `notapad_app/ui_engine.py` + `notapad.py`
The custom menu (Frames placed with `.place()`) requires ~200 lines of workarounds for
click-outside-to-close, hover-to-switch, and theme synchronization. It has no keyboard
navigation (see L8). Tk's native `Menu` widget provides all of this for free and would
eliminate the complexity entirely. The visual style can still be themed via `tk.Menu`'s
configuration options.

### A4 вЂ” Formalize the settings schema
**File:** `notapad_app/settings.py`
`SettingsManager` uses a plain dict with string keys. A `dataclasses.dataclass` or
`typing.TypedDict` would catch key-name typos at development time, provide IDE
autocomplete, and make the schema self-documenting. Example:
```python
from dataclasses import dataclass, field
@dataclass
class NotapadSettings:
    theme_mode: str = "system"
    word_wrap: bool = True
    font_family: str = "Consolas"
    font_size: int = 11
    recent_files: list = field(default_factory=list)
    ...
```

### A5 вЂ” Move debounced settings saves to root.after() instead of threading.Timer
**File:** `notapad_app/settings.py`
The current `threading.Timer(2.0, ...)` approach introduces a daemon thread for every
debounced save operation. All Tk operations must be on the main thread anyway, so there
is no benefit to using a background thread here. Replacing with
`self._root.after(2000, self.save)` eliminates the thread entirely and removes the
thread-safety concern (M10).
